# 知识蒸馏 (Knowledge Distillation)

## 核心思想

**大模型当老师，教小模型"类间关系"，而不仅仅是标准答案。**

传统训练：学生只看"正确答案"（hard label = 这是猫）。
蒸馏训练：老师告诉你"猫 90%、老虎 8%、豹子 2%"——这些概率包含了**暗知识（dark knowledge）**：猫和老虎更像，和汽车完全不像。

```
真实标签 (Hard Label)： 猫 = [1, 0, 0, 0, 0]   ← 只有对/错
教师软标签 (Soft Label)：猫 = [0.85, 0.10, 0.03, 0.01, 0.01]
                               猫    狗   老虎  豹    汽车
                                        ↑ 老虎跟猫比较像
```

---

## 蒸馏不是替代真实标签，而是补充

```python
总损失 = α × KL(学生预测, 教师软标签) + (1-α) × CrossEntropy(学生预测, 真实标签)
              ↑ 蒸馏损失：学老师的判断方式               ↑ 原始损失：学正确答案
```

---

## 两种蒸馏方式

| 蒸馏方式 | 核心思想 | 损失函数 |
|---------|---------|---------|
| **基于响应（Response-Based）** | 学生模仿教师的最终预测（软标签） | `KLDivLoss`（KL 散度） |
| **基于特征（Feature-Based）** | 学生模仿教师的中间层特征图 | `MSELoss`（均方误差） |

实践中通常**两者结合**使用。

---

## 关键参数：温度 (Temperature)

温度越高，概率分布越"软"，类间关系暴露得越充分：

```python
# T=1：  [0.90, 0.08, 0.02]  ← 比较"硬"，接近原始预测
# T=5：  [0.60, 0.25, 0.15]  ← "软"了，次要概率被放大
# T=20： [0.40, 0.33, 0.27]  ← 很软，类别间差异被抹平

soft_teacher = F.softmax(teacher_logits / T, dim=1)  # 温度越高越软
```

# YOLO 目标检测蒸馏（完整代码）

## 第1步：定义蒸馏损失函数

```python
import torch
import torch.nn as nn
import torch.nn.functional as F
from ultralytics import YOLO
from ultralytics.models.yolo.detect import DetectionTrainer
from ultralytics.nn.tasks import DetectionModel

class KDLoss:
    def __init__(self, student_model, teacher_model, kd_weight=0.5, temp=5.0):
        """
        Args:
            kd_weight: 蒸馏损失权重
            temp: 蒸馏温度，越大分布越"软"
        """
        self.teacher = teacher_model
        self.student = student_model
        self.kd_weight = kd_weight
        self.temp = temp

        self.ce_loss = nn.CrossEntropyLoss()           # 硬标签损失
        self.kl_loss = nn.KLDivLoss(reduction='batchmean')  # 软标签损失
        self.mse_loss = nn.MSELoss()                   # 特征损失

    def __call__(self, student_outputs, teacher_outputs, gt_labels,
                 student_feats, teacher_feats):
        # 1. 硬标签损失：学生学真实标注
        loss_hard = self.ce_loss(student_outputs, gt_labels)

        # 2. 软标签损失：学生模仿教师的最终预测
        soft_teacher = F.softmax(teacher_outputs / self.temp, dim=1)
        log_soft_student = F.log_softmax(student_outputs / self.temp, dim=1)
        loss_soft = self.kl_loss(log_soft_student, soft_teacher) * (self.temp ** 2)

        # 3. 特征损失：学生学习教师中间层特征图（P3, P4, P5）
        loss_feat = 0
        for s_feat, t_feat in zip(student_feats, teacher_feats):
            loss_feat += self.mse_loss(s_feat, t_feat)

        # 总损失
        total_loss = loss_hard + self.kd_weight * (loss_soft + loss_feat)
        return total_loss
```

## 第2步：自定义训练器

```python
class KDTrainer(DetectionTrainer):
    def __init__(self, cfg=None, overrides=None,
                 teacher_model_path='yolo11m.pt', kd_weight=0.5):
        super().__init__(cfg, overrides)

        # 加载教师模型并冻结
        self.teacher = YOLO(teacher_model_path)
        self.teacher.model.eval()
        for p in self.teacher.model.parameters():
            p.requires_grad = False

        self.kd_loss = KDLoss(self.model, self.teacher.model, kd_weight=kd_weight)

        # 注册钩子捕获中间层特征
        self.student_feats = []
        self.teacher_feats = []
        self._register_hooks()

    def _register_hooks(self):
        """为 P3, P4, P5 层注册前向钩子"""
        target_layers = [9, 12, 15]  # 示例索引，实际需按模型结构调整

        def hook_fn(module, input, output, storage):
            storage.append(output.detach())

        for layer_idx in target_layers:
            self.model.model.model[layer_idx].register_forward_hook(
                lambda m, i, o: hook_fn(m, i, o, self.student_feats))
            self.teacher.model.model[layer_idx].register_forward_hook(
                lambda m, i, o: hook_fn(m, i, o, self.teacher_feats))

    def loss(self, batch, preds=None):
        self.student_feats.clear()
        self.teacher_feats.clear()

        if preds is None:
            preds = self.model(batch['img'])

        with torch.no_grad():
            teacher_preds = self.teacher.model(batch['img'])

        total_loss = self.kd_loss(
            preds, teacher_preds, batch['cls'],
            self.student_feats, self.teacher_feats
        )

        return total_loss, torch.tensor([0.0])
```

## 第3步：启动蒸馏训练

```python
if __name__ == '__main__':
    student_model = DetectionModel('yolo11s.yaml', ch=3, nc=80)

    args = {
        'model': student_model,
        'data': 'coco8.yaml',
        'epochs': 50,
        'batch': 16,
        'imgsz': 640,
        'device': 0,
    }

    trainer = KDTrainer(
        overrides=args,
        teacher_model_path='yolo11m.pt',
        kd_weight=0.5,
    )
    trainer.train()
```

# 大语言模型（LLM）蒸馏

## YOLO 蒸馏 vs LLM 蒸馏 对比

| 组件 | YOLO 检测 | 大语言模型 |
|------|----------|-----------|
| **输出形式** | 分类概率 + 回归框 | 序列中每个 token 的概率分布 |
| **硬标签损失** | `CrossEntropy(预测类别, 真实类别)` | `CrossEntropy(预测token, 训练文本真实token)` |
| **软标签损失** | `KL(学生分类分布, 教师分类分布)` | `KL(学生每个token的分布, 教师分布)` ← 在整条序列上求和 |
| **特征损失** | `MSE(学生FPN特征图, 教师FPN特征图)` | `MSE(学生hidden states, 教师hidden states)` ← 需维度对齐 |
| **温度作用** | 一般重要 | **非常关键**（词表几万~几十万，温度暴露语义近邻） |

## LLM 蒸馏核心代码

```python
for batch in dataloader:
    # 教师模型前向（冻结）
    with torch.no_grad():
        teacher_outputs = teacher(input_ids)  # [B, seq_len, vocab_size]
        teacher_hidden = teacher.last_hidden_state

    # 学生模型前向
    student_outputs = student(input_ids)      # [B, seq_len, vocab_size]
    student_hidden = student.last_hidden_state

    # 1. 硬标签损失
    loss_hard = F.cross_entropy(
        student_outputs.view(-1, vocab_size),  # [B*seq_len, vocab_size]
        target_ids.view(-1)                    # 真实下一个 token
    )

    # 2. 软标签损失：每个 token 位置都算
    loss_soft = F.kl_div(
        F.log_softmax(student_outputs / T, dim=-1),
        F.softmax(teacher_outputs / T, dim=-1),
        reduction='batchmean'
    ) * (T ** 2)

    # 3. 特征损失（维度不同需投影）
    # 学生 hidden [B, seq, 2048] → 投影到教师维度 [B, seq, 4096]
    student_proj = nn.Linear(student_hidden_dim, teacher_hidden_dim)
    loss_feat = F.mse_loss(
        student_proj(student_hidden),
        teacher_hidden
    )

    loss = loss_hard + alpha * loss_soft + beta * loss_feat
    loss.backward()
```

## LLM 蒸馏的特殊问题

### 1. 温度为什么更关键

LLM 词表大（3万~15万），教师预测中语义相近的词会有不小概率：
```
输入: "我的宠物是"
教师: [猫: 0.45, 狗: 0.30, 宠物: 0.08, 动物: 0.05, ...]
        ↑ 正确答案      ↑ 这些"第二名"只存在于软标签中，hard label 完全丢弃了
```

软标签让学生在犯错时知道错得有多离谱——预测"狗"比预测"汽车"好得多。

### 2. 特征维度对齐

教师和学生 hidden dim 通常不同（7B 老师的 4096 vs 1B 学生的 2048），需要加投影层：

```python
# 方法1：线性投影
student_proj = nn.Linear(student_dim, teacher_dim).to(device)

# 方法2：MLP
student_proj = nn.Sequential(
    nn.Linear(student_dim, student_dim * 2),
    nn.ReLU(),
    nn.Linear(student_dim * 2, teacher_dim),
).to(device)
```

### 3. 生成式蒸馏（最流行的方法）

与其逐 token 对齐，不如让教师直接生成高质量训练数据：

```python
# Step 1: 用教师模型生成回答
for prompt in dataset:
    answer = teacher.generate(prompt, max_length=500)
    save(prompt, answer)  # → distilled_dataset.jsonl

# Step 2: 学生直接用这些数据做 SFT
student.train(distilled_dataset)  # 标准的监督微调，无需额外蒸馏代码
```

这是目前最火的 LLM 蒸馏方法——简单、粗暴、效果好。DeepSeek-R1、Alpaca 系列都用的类似思路。

## 进阶方法

| 方法 | 核心思想 | 适用场景 |
|------|---------|---------|
| **CWD** | 按通道归一化特征图后再计算损失，避免尺度不匹配 | 通用，效果好 |
| **MGD** | 让学生只学教师的部分"遮罩"特征，迫使其掌握本质信息 | 追求极致性能 |
| **BCKD** | 双分支结构，同时迁移分类和回归的知识 | 目标检测专用 |
| **三明治结构** | 多个学生夹在老师中间，层层传递知识 | 模型需要压到极小时 |

---

## 常见问题

| 问题 | 原因 | 解决方案 |
|------|------|---------|
| **精度不升反降** | 蒸馏权重太大，干扰正常学习 | 降低 `kd_weight`；前几个 epoch 不加蒸馏（预热） |
| **训练变慢** | 每次迭代跑两次前向（老师+学生） | 数据集小时预缓存教师软标签；教师用 FP16 |
| **架构不同（如 YOLO 教 ResNet）** | 中间层维度、语义不匹配 | 响应式蒸馏仍有效（输出维度相同即可）；特征蒸馏需加适配层 |
| **老师太强，学生跟不上** | 学生容量不足以拟合教师分布 | 降低温度 T，让分布更"硬"；或分阶段蒸馏 |
